Capitalization

We use different capitalization styles for different types of content to improve scannability, organize information, and guide users to key actions.

When building new Firefox products, we also align capitalization practices with platform conventions to match user expectations.

Capitalization Styles

Title Case

For title case, most words are capitalized, with some exceptions.

Capitalize the first letter of nouns, adjectives, verbs, adverbs, and pronouns.

Capitalize the first and last words of the text element.

Capitalize the second word in a hyphenated compound if it would normally be capitalized as a single word. For example, you would capitalize both words in “Quick-Search Options,” but you would only capitalize “Add” in “Add-on Choices.”

Lowercase articles (“a,” “an,” “the”) or prepositions with fewer than five letters (“to,” “on,” “at,” “by,” “from”) unless they’re the first or last words of the text element.

Sentence Case

For sentence case, most words are lowercase, with some exceptions.

Only capitalize the first word of the text element.

Capitalize any other words you would normally capitalize, like proper nouns.

All Caps

For all caps, every word in the text element is capitalized.

Platform-specific Styles

Title Case on Desktop

Use title case for menu labels, buttons, navigation, and section headers.

Sentence Case on Desktop

Use sentence case for all body copy, image captions, and secondary, editorial subheadings.

Allcaps on Desktop

In general, do not use all caps in Firefox for desktop. In rare cases, all caps may be used to visually distinguish a secondary, less important text element.

Title Case on Android

In general, do not use title case in Firefox for Android.

Sentence Case on Android

Use sentence case for all text elements except buttons.

Allcaps on Android

Use all caps for buttons.

For more information, see the Writing section of the Google-maintained Material Design Style Guide.

Title Case on iOS

Use title case for menu labels, buttons, navigation, and section headers.

Sentence Case on iOS

In general, use sentence case for all other text elements, including image captions.

Allcaps on iOS

Use all caps for subheadings in app Settings.

For more information, see Apple’s Human Interface Guidelines.
