Errors

Errors (or error condition) occur when an the software fails to complete an expected action. Every error becomes a point of friction for the user.

Usage

Error messages provide short and relevant information that adds context when a user encounters a problem or issue. Whenever possible, provide the user with an action that helps them address the error. Errors are usually place close to the context.

It's important to distinguished two different kind of error:

  • User input errors
  • System errors

User input errors

System errors occurs when Firefox fails for some reasons most likely not related to the user.

The most common cases are:

  • Misspelled errors
  • Connettivity issues
  • Failure to load

System errors

User input error occurs when Firefox doesn't understand user inputs.

The most common cases are:

  • Incomplete form
  • Over/under character or word count
  • Incompatible values

Style

Icon

The icon used for indicate errors is an exclamation mark enclosed within a triangle, usually white on a red background, or red.

Colors

Red is the dedicated color for errors states. Warning text should be legible, with noticeable contrast against its background color.

Background

Red 50 #ff0039, or transparent

Content

White #ffffff, or Red 60 #d70022

Copy rules

In case of an error, the following principles can help you to write helpful content which empowers users to recover quickly.

  • Clearly explain what went wrong and how to fix it.
  • Be short and concise, no more than a single sentence.
  • Use passive voice. You don't want to blame the users for the error.
